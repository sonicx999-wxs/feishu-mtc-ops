### @步骤1: 创建文档（含降级）

<!-- @类型: 操作步骤（含降级） -->

- @动作: 检查返回 `ok: true`，获取 `doc_id` 和 `doc_url`
- @动作: **降级1**：CLI失败时使用 Python脚本 `python3 scripts/create_doc.py --title "标题" --content "内容"`

### @步骤2: 获取/更新文档

<!-- @类型: 操作步骤 -->

- @动作: 全量覆盖：`lark-cli docs +update --doc "文档ID" --mode overwrite --markdown "新内容"`
- @动作: 替换章节：`lark-cli docs +update --doc "文档ID" --mode replace_range --selection-by-title "## 章节" --markdown "新内容"`

<!-- @类型: 操作步骤 -->
<!-- @优先级: 可选 -->
<!-- @验证点: 大文档能被正确全量更新 -->
<!-- @验证方式: 检查文档内容完整性和格式正确性 -->

- @动作: 注意：表格不可切割，必须保持完整或拆为多个独立表格

### @步骤6: 用户 ID 解析（重要）

<!-- @类型: 操作步骤 -->
<!-- @优先级: 必须 -->
<!-- @验证点: 正确的open_id被使用 -->
<!-- @验证方式: 检查open_id来源 -->

> ⚠️ **关键说明**：open_id 跨应用不通用！lark-cli 返回的是 SOLO 平台应用下的 open_id，MCP 应用无法识别

**SOLO MTC 环境双应用隔离问题**：
| 属性 | lark-cli（平台预设） | MCP 服务器（用户配置） |
|------|---------------------|----------------------|
| 应用 ID | `cli_a965abcba6fadbd3` | `cli_a9711cf63678dcc7` |
| 用户 open_id | 不同 | 不同 |
| 用户 union_id | 相同 | 相同 |
| 租户 | 相同 | 相同 |

**获取用户 open_id 的推荐流程**：
```
1. 尝试通过群成员列表获取（推荐）
   └── 调用 GET /im/v1/chats/{chat_id}/members?member_id_type=open_id
   └── 在返回结果中匹配用户名获取 open_id
   └── ✅ 成功 → 缓存结果，后续直接使用

2. 如果无法获取，生成群聊邀请链接
   └── 调用 POST /im/v1/chats/{chat_id}/link
   └── 用户点击加入后，回到步骤 1
```

> 💡 **最佳实践**：应用身份创建的资源应由应用身份管理，user 身份无法操作。

## 注意事项

1. **MTC模式固定 user身份**，不要添加 `--as user`或 `--as bot`
2. **lark-cli 无法发送聊天消息**，必须使用脚本或 MCP工具
3. user身份创建的文档归用户，useUAT: false创建的文档归应用
4. 使用应用身份创建文档后，必须为用户添加权限
5. 检查返回 JSON中的 `ok`字段或 `error`字段
6. 异步任务使用 `drive +task_result`查询结果
7. **禁止硬编码凭证**，必须从 MCP 配置读取
8. **群聊搜索使用 GET /im/v1/chats**，不要使用 search 接口
9. **执行前先进行环境预检**，根据环境类型选择合适的工具和降级路径

### 错误码速查

| 错误码 | 含义 | 处理方式 |
|--------|------|---------|
| 230027 | 权限不足 | 用户身份无权限，请使用应用身份脚本发送消息 |
| 232011 | Operator not in chat | 操作者不在目标群聊中，请先加入群聊或使用应用身份 |
| 9499 | 请求格式错误 | 检查 content 字段 JSON 嵌套是否正确 |
| 99991663 | 群聊不存在 | 检查 chat_id 是否正确 |
| 99991661 | 无权限发送 | 检查应用是否被添加到群聊 |
| 99992361 | open_id cross app | **关键**：open_id 跨应用不通用，需通过群成员列表获取 MCP 应用下的 open_id |
| 99992364 | user id cross tenant | 实际是 open_id 跨应用问题，请使用群成员列表获取正确 open_id |
| 1063003 | Invalid operation | 用户身份创建的文档无法通过 API 添加权限，请使用应用身份创建文档 |
| 1063002 | Permission denied | 应用身份无权限操作该文档 |
| 230001 | invalid receive_id | 群聊 ID 不存在或格式错误 |
